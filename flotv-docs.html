FloatTV Documentation

This page documents the current `flotv.html` build, the UI changes made during iteration, and the expected usage flow for floating video panels.

File: flotv.html Primary use: floating video panels Current source mode: embed/live links

What Changed

  • The top hero section was converted into a narrow left sidebar so the canvas gets nearly the full screen width.
  • The sidebar now holds the title, URL input, `Float Channel`, `Stack Tiles`, and `Clear All` controls.
  • A `Fit Max` button was added to pack as many floating channels into the canvas as the available space allows.
  • Floating windows remain draggable and resizable, but the page now prioritizes workspace area over controls.
  • The stray literal EOF text accidentally written into the page was removed.

Playback Behavior

  • Clicking `Float Channel` now appends playback parameters so the panel attempts autoplay=1, mute=1, and playsinline=1.
  • Direct video and HLS sources use a real browser video element so mute, autoplay, ready state, and stream delay can be observed directly.
  • The iframe allow list was expanded to include autoplay and picture-in-picture behavior.
  • The delay badge now shows LIVE near the live edge and values such as -10 sec when the stream is behind.
  • Autoplay still depends on the provider and browser policy; some sources will ignore it or block playback.

How To Use FloatTV

  1. Open `flotv.html` in the browser.
  2. Paste a supported embed-capable stream or channel URL into the sidebar input.
  3. Click `Float Channel` to create a new floating panel.
  4. Drag the panel header to move it around the canvas.
  5. Drag the bottom-right resize handle to change the panel size.
  6. Use `Stack Tiles` to gather all active panels into a centered stack.
  7. Use `Hide` on a panel to remove just that one, or `Clear All` to reset the canvas.

Supported Inputs

  • YouTube channel URLs using the /channel/ format are converted to the live stream embed pattern.
  • YouTube watch URLs with a v= parameter are converted into embed URLs.
  • Direct .m3u8 links are treated as HLS streams and routed through browser-aware playback logic.
  • Direct embed links and other raw URLs are passed through with autoplay/mute parameters appended.
  • If the upstream provider blocks embedding, the panel will still render but playback may fail.

Known Limits

  • Some YouTube videos and channels reject embedding or autoplay because of origin, policy, or player restrictions.
  • The fullscreen toggle button currently only toggles a class and does not yet apply a dedicated fullscreen layout.
  • Panels do not persist between reloads yet.
  • Live delay and status are reliable for direct video/HLS playback, but iframe-based providers still expose limited runtime state.

Issue Resolution Log

  • The TRT World stream issue exposed a browser mismatch: direct native HLS playback returned `Unsupported` on non-Safari browsers.
  • A temporary fallback path was tried, then reverted when it broke a previously working stream path.
  • The final fix uses browser-aware routing: Safari-like browsers use native HLS, while Chrome/Edge-style browsers use hls.js for .m3u8 streams.
  • The debug panel was added first as a canvas overlay, then moved into the left sidebar so the canvas stayed clean while diagnostics remained visible.
  • The debug output now includes source type, mode, status, delay, paused/muted state, ready/network state, current time, media error, and diagnostic text.

Implementation Notes

  • The page uses a two-column layout: a constrained sidebar on the left and a large relative-positioned canvas on the right.
  • Each floating panel is an absolutely positioned DOM node with a draggable header and resize handle.
  • URL conversion is handled in JavaScript before the player is created, so playback flags and source classification are injected centrally.
  • HLS playback is now browser-aware: native in Safari-like browsers and hls.js in non-Safari browsers.
  • The current implementation favors lightweight provider handling while exposing enough diagnostics to troubleshoot stream issues quickly.
